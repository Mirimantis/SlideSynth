# BACKLOG 8.19 ‚Äî Rename Key "None" ‚Üí "Chromatic" + new true-None mode

## Context

The Key dropdown's current "None" option is a misnomer: it actually renders all 12 chromatic semitones and Y-snaps to the nearest semitone. We want:

1. **Rename** the existing "None" ‚Üí **Chromatic** (so the label matches the behavior).
2. **Add a new "None"** option that hides every default pitch line on the staff. The implicit chromatic-semitone Y-snap goes away (cursor Y is free between guides), but every other Y-snap target keeps working: user-defined Y-guides (8.7), Harmonic Prism projection echoes, and any existing point/curve snapping. None mode just means "no background staff lines and no auto-pull to integer semitones" ‚Äî explicit targets are unaffected.

Why: a user who has set up custom pitch guides via 8.7 wants a clean canvas with only their guides ‚Äî no default 12-semitone background. Today there's no way to suppress the default pitch lines without disabling all snap.

## Design

The dropdown represents three modes; today they're encoded as `scaleRoot: number | null` (number = scale tone, null = chromatic). Add a sibling boolean `hidePitchLines: boolean` to `SnapSettings`. The toolbar treats Key as one tri-state selector internally encoded as two fields:

| Dropdown selection | `scaleRoot`  | `hidePitchLines` |
|--------------------|--------------|------------------|
| Chromatic (default)| `null`       | `false`          |
| None               | `null`       | `true`           |
| C, D, E‚Ä¶           | `0..11`      | `false`          |

Why a boolean flag instead of widening `scaleRoot` to a string union:
- Zero migration: existing composition files with `scaleRoot: null` continue to mean Chromatic.
- Keeps `scaleRoot` as `number | null` (used for indexing scale math).
- Picking a scale tone in the UI forces `hidePitchLines = false` (single source of truth at the toolbar layer).
